Control Barrier Functions (CBFs)

A barrier enforces a safety constraint \(h(q) \ge 0\) by adding inequality rows to the QP that prevent the robot from entering unsafe regions.

CBF inequality

For a barrier function \(h : \mathcal{Q} \to \mathbb{R}\):

\[ \nabla_q h(q)^\top \Delta q \;\ge\; -\gamma_b\; h(q) \]

or equivalently (in \(G\,\Delta q \le h\) form):

\[ -\nabla_q h(q)^\top \Delta q \;\le\; \gamma_b\; h(q) \]

where \(\gamma_b > 0\) is the barrier gain .

Safe displacement objective

In addition to the constraint, barriers add a secondary objective that pushes the robot away from the boundary:

\[ \min_{\Delta q}\; \tfrac{1}{2}\,\kappa \left\|\nabla_q h(q)^\top \Delta q + \gamma_b\; h(q)\right\|^2 \]

where \(\kappa\) is the safe displacement gain . When \(\kappa = 0\) only the hard constraint is active; when \(\kappa > 0\) the solver also actively moves the robot away from the unsafe boundary.

Planned barrier types

PositionBarrier

Bound the Cartesian position of a frame along selected axes.

\[ p_{\min}^{(i)} \le p_{\text{frame}}^{(i)} \le p_{\max}^{(i)}, \quad i \in \{\text{x}, \text{y}, \text{z}\} \]

Parameter Type Description
frame string Frame to monitor
indices list of int Position axes (0=x, 1=y, 2=z)
p_min / p_max vector Minimum / maximum position bounds
gain float Barrier gain \(\gamma_b\)
safe_displacement_gain float \(\kappa\)

BodySphericalBarrier

Maintain minimum distance between two frames (collision avoidance between specific body pairs).

\[ h(q) = \|p_A(q) - p_B(q)\| - d_{\min} \]

Parameter Type Description
frames (string, string) Two frame names
d_min float Minimum distance (m)
gain float Barrier gain \(\gamma_b\)
safe_displacement_gain float \(\kappa\) (default 3.0)

SelfCollisionBarrier

Monitor multiple collision pairs from the Pinocchio geometry model and keep them above a minimum distance.

\[ h_k(q) = d_k(q) - d_{\min}, \quad k = 1, \dots, n_{\text{pairs}} \]

Parameter Type Description
n_collision_pairs int Number of closest pairs to monitor
d_min float Minimum distance (m), default 0.02
gain float Barrier gain \(\gamma_b\)
safe_displacement_gain float \(\kappa\) (default 1.0)

How barriers fit into the QP

Once implemented, barriers will contribute:

  1. Inequality rows to the constraint matrix \((G, h)\), alongside velocity and configuration limits.
  2. Objective terms to \((P, c)\) via the safe-displacement cost (analogous to task contributions but with barrier-specific weights).

The overall QP structure remains unchanged — barriers are additive.

Common barrier parameters

Parameter Symbol Effect
gain \(\gamma_b\) How aggressively the barrier is enforced. Higher = stricter.
safe_displacement_gain \(\kappa\) Strength of the "push away" objective. 0 = constraint only.
d_min \(d_{\min}\) Minimum allowable distance/clearance.
