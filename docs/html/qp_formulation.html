QP Formulation

Torq solves differential inverse kinematics as a Quadratic Program (QP) at every control tick. This page derives the complete formulation, showing how tasks, limits, and barriers compose into a single optimisation problem.

See Conventions and Notation for notation and Tasks, Limits, Barriers for the per-component derivations.

Overview

Given the current robot configuration \(q\), the IK solver finds a displacement \(\Delta q \in \mathbb{R}^{n_v}\) (tangent-space velocity \(\times\,\Delta t\)) that best satisfies all active tasks while respecting physical limits and safety barriers:

\[ \boxed{ \begin{aligned} \min_{\Delta q} \quad & \tfrac{1}{2}\,\Delta q^\top P\,\Delta q + c^\top \Delta q \\ \text{s.t.} \quad & G\,\Delta q \le h \end{aligned} } \]

where:

Component Source Role
\(P\) (positive semi-definite) Tasks + barrier objectives + Tikhonov damping Quadratic cost on \(\Delta q\)
\(c\) Tasks + barrier objectives Linear cost on \(\Delta q\)
\(G, h\) Limits + barrier inequalities Hard inequality constraints

Per-task contribution

Every task \(i\) defines:

Symbol Meaning Shape Contribution
\(e_i(q)\) Task error \(k_i \times 1\) Drives the objective gradient
\(J_i(q)\) Task Jacobian \(k_i \times n_v\) Maps \(\Delta q\) to task-space changes
\(w_i\) Cost (weight) vector \(k_i \times 1\) Scales error and Jacobian components
\(\alpha_i\) Gain \(\in (0, 1]\) scalar Controls convergence speed
\(\mu_i\) LM damping scalar Regularises near singularities

Weight matrix

The diagonal weight matrix \(W_i = \operatorname{diag}(w_i)\) scales each component of the error independently. For FrameTask the 6-D weight vector is \([w_{\text{pos}}^{(3)},\; w_{\text{ori}}^{(3)}]\).

Weighted quantities

\[ \bar{J}_i = W_i \, J_i, \qquad \bar{e}_i = W_i \left(-\alpha_i \, e_i\right) \]

The task wants \(\bar{J}_i \, \Delta q \approx \bar{e}_i\), which in least-squares form is:

\[ \min_{\Delta q}\; \tfrac{1}{2}\left\|\bar{J}_i\,\Delta q - \bar{e}_i\right\|^2 \]

Levenberg-Marquardt damping

When the target is unreachable the error can be large and the solver may produce huge velocities. LM damping adds a configuration-space regularisation proportional to the task's own residual:

\[ \mu_i^{\text{eff}} = \mu_i \cdot \bar{e}_i^\top \bar{e}_i \]

This is adaptive: the damping is strong when the residual is large (unreachable target) and vanishes when the error is small.

Per-task Hessian and gradient

\[ H_i = \bar{J}_i^\top \bar{J}_i + \mu_i^{\text{eff}}\, I_{n_v}, \qquad c_i = -\bar{J}_i^\top \bar{e}_i \]

Composing multiple tasks

Because tasks live in the same \(\Delta q\) space, the QP Hessian and gradient are simply summed:

\[ P_{\text{tasks}} = \sum_{i} H_i, \qquad c_{\text{tasks}} = \sum_{i} c_i \]

The relative importance of each task is encoded in its weight vector \(w_i\). Higher cost → the solver prioritises that task more aggressively.

For the default built-in tasks:

Task Typical cost Effect
FrameTask 1.0 Primary objective: track end-effector
PostureTask \(10^{-3}\) Regularisation: stay near home
DampingTask \(10^{-4}\) Smoothing: minimise motion

Tikhonov (solver) damping

A small global regularisation \(\lambda\) (default \(10^{-12}\)) is added to the diagonal of \(P\) to guarantee positive definiteness even when tasks do not fully constrain all DOFs:

\[ P \leftarrow P + \lambda \, I_{n_v} \]

Unlike LM damping (per-task, error-dependent), Tikhonov damping is a constant background term that ensures the QP is always solvable.

Inequality constraints from limits

Each Limit produces rows \((G_j, h_j)\) such that \(G_j\,\Delta q \le h_j\).

All limit rows are stacked vertically:

\[ G_{\text{limits}} = \begin{bmatrix} G_1 \\ G_2 \\ \vdots \end{bmatrix}, \qquad h_{\text{limits}} = \begin{bmatrix} h_1 \\ h_2 \\ \vdots \end{bmatrix} \]

See Limits for the exact derivation of each limit type.

Barrier contributions

Each Barrier contributes to both the objective and the constraints:

Barrier inequality rows

The CBF condition for barrier \(j\):

\[ G_j = -\frac{1}{\Delta t}\,\frac{\partial h_j}{\partial q}, \qquad h_j^{\text{rhs}} = \alpha_j \cdot g\!\left(h_j(q)\right) \]

where \(g(\cdot)\) is the barrier's gain function (default: identity).

Barrier rows are stacked alongside limit rows:

\[ G = \begin{bmatrix} G_{\text{limits}} \\ G_{\text{barriers}} \end{bmatrix}, \qquad h = \begin{bmatrix} h_{\text{limits}} \\ h_{\text{barriers}} \end{bmatrix} \]

Barrier objective terms

If the safe-displacement gain \(\kappa_j > 0\):

\[ H_j^{\text{barrier}} = \frac{\kappa_j}{\|J_j\|_F^2}\,I_{n_v}, \qquad c_j^{\text{barrier}} = -\frac{\kappa_j}{\|J_j\|_F^2}\,\Delta q_{\text{safe},j} \]

These are summed into the QP objective alongside task contributions.

The complete QP

Putting it all together:

\[ \boxed{ \begin{aligned} \min_{\Delta q} \quad & \tfrac{1}{2}\,\Delta q^\top \underbrace{\left[ \lambda I + \sum_{i \in \text{tasks}} H_i + \sum_{j \in \text{barriers}} H_j^{\text{barrier}} \right]}_{P}\,\Delta q + \underbrace{\left[ \sum_{i \in \text{tasks}} c_i + \sum_{j \in \text{barriers}} c_j^{\text{barrier}} \right]^\top}_{c^\top}\,\Delta q \\[8pt] \text{s.t.} \quad & \underbrace{\begin{bmatrix} G_{\text{limits}} \\ G_{\text{barriers}} \end{bmatrix}}_{G}\,\Delta q \;\le\; \underbrace{\begin{bmatrix} h_{\text{limits}} \\ h_{\text{barriers}} \end{bmatrix}}_{h} \end{aligned} } \]

OSQP solver

Torq uses OSQP via OsqpEigen. OSQP expects the standard form:

\[ \min_x \;\tfrac{1}{2}\,x^\top P\,x + q^\top x \quad\text{s.t.}\quad l \le A\,x \le u \]

The mapping from Torq's QPProblem is:

Torq OSQP
\(P\) (Hessian) \(P\) (upper triangle, sparse)
\(c\) \(q\)
\(G\) \(A\)
\(-\infty\) \(l\)
\(h\) \(u\)

The solver is warm-started across ticks: the sparsity pattern is set once and only the numerical values are updated on subsequent solves. When there are no inequality constraints ( \(m = 0\)), Torq uses a direct LDLT decomposition instead of OSQP for better performance.

Manifold-aware integration

The solved \(\Delta q\) is a tangent vector. For joints with manifold structure (e.g. floating-base quaternions) Torq uses Pinocchio's pinocchio::integrate instead of simple addition:

\[ q_{\text{new}} = q \oplus \Delta q \]

For revolute and prismatic joints \(\oplus\) reduces to ordinary addition, so there is no overhead for fixed-base arms. For floating-base robots, the quaternion component is updated using the exponential map, maintaining the unit-norm constraint and avoiding gimbal lock.

See Conventions and Notation for a full explanation of manifold operations.

Velocity output

The QP solves for a displacement \(\Delta q\). The corresponding generalised velocity is:

\[ v = \frac{\Delta q}{\Delta t} \]

This velocity is returned to the caller and used by limits (which scale their bounds by \(\Delta t\)) and by Configuration::integrate().

Summary of the assembly pipeline

  1. Tikhonov: \(P = \lambda I\), \(c = 0\).
  2. Tasks: For each task, compute \((H_i, c_i)\) → \(P \mathrel{+}= H_i\), \(c \mathrel{+}= c_i\).
  3. Barrier objectives: For each barrier with \(\kappa > 0\), compute \((H_j, c_j)\) → \(P \mathrel{+}= H_j\), \(c \mathrel{+}= c_j\).
  4. Limits: For each limit, compute \((G_j, h_j)\) → stack into \((G, h)\).
  5. Barrier inequalities: For each barrier, compute \((G_j, h_j)\) → stack into \((G, h)\).
  6. Solve: If \(m > 0\) use OSQP; else use direct LDLT.
  7. Integrate: \(q_{\text{new}} = q \oplus \Delta q\).
