QP Formulation

Torq solves differential inverse kinematics as a Quadratic Program (QP) at every control tick. This page derives the complete formulation — how tasks, limits, and damping compose into a single optimisation problem.

Overview

Given the current robot configuration \(q\), the IK solver finds a displacement \(\Delta q \in \mathbb{R}^{n_v}\) (tangent-space velocity \(\times\,\Delta t\)) that best satisfies all active tasks while respecting physical limits:

\[ \begin{aligned} \min_{\Delta q} \quad & \tfrac{1}{2}\,\Delta q^\top P\,\Delta q + c^\top \Delta q \\ \text{s.t.} \quad & G\,\Delta q \le h \end{aligned} \]

where \(P\) (positive semi-definite) and \(c\) accumulate task objectives, and the rows of \((G, h)\) encode limit constraints.

Per-task contribution

Every task \(i\) defines:

Symbol Meaning Shape
\(e_i(q)\) Task error \(k_i \times 1\)
\(J_i(q)\) Task Jacobian \(k_i \times n_v\)
\(w_i\) Cost (weight) vector \(k_i \times 1\)
\(\alpha_i\) Gain \(\in (0, 1]\) scalar
\(\mu_i\) Levenberg–Marquardt damping scalar

Weight matrix

The diagonal weight matrix \(W_i = \operatorname{diag}(w_i)\) scales each component of the error independently. For FrameTask the 6-D weight vector is \([w_{\text{pos}}^{(3)},\; w_{\text{ori}}^{(3)}]\).

Weighted quantities

\[ \bar{J}_i = W_i \, J_i, \qquad \bar{e}_i = W_i \left(-\alpha_i \, e_i\right) \]

The task wants \(\bar{J}_i \, \Delta q \approx \bar{e}_i\), which in least-squares form is:

\[ \min_{\Delta q}\; \tfrac{1}{2}\left\|\bar{J}_i\,\Delta q - \bar{e}_i\right\|^2 \]

Levenberg–Marquardt damping

When the target is unreachable the error can be large and the solver may produce huge velocities. LM damping adds a configuration-space regularisation proportional to the task's own residual:

\[ \mu_i^{\text{eff}} = \mu_i \cdot \bar{e}_i^\top \bar{e}_i \]

Per-task Hessian and gradient

\[ H_i = \bar{J}_i^\top \bar{J}_i + \mu_i^{\text{eff}}\, I_{n_v}, \qquad c_i = -\bar{J}_i^\top \bar{e}_i \]

Composing multiple tasks

Because tasks live in the same \(\Delta q\) space, the QP Hessian and gradient are simply summed:

\[ P = \sum_{i} H_i, \qquad c = \sum_{i} c_i \]

The relative importance of each task is encoded in its weight vector \(w_i\). Higher cost → the solver prioritises that task more aggressively.

Tikhonov (solver) damping

A small global regularisation \(\lambda\) (default \(10^{-12}\)) is added to the diagonal of \(P\) to guarantee positive definiteness even when tasks do not fully constrain all DOFs:

\[ P \leftarrow P + \lambda \, I_{n_v} \]

Unlike LM damping (per-task, error-dependent), Tikhonov damping is a constant background term.

Inequality constraints from limits

Each Limit produces rows \((G_j, h_j)\) such that \(G_j\,\Delta q \le h_j\).

All limit rows are stacked vertically:

\[ G = \begin{bmatrix} G_1 \\ G_2 \\ \vdots \end{bmatrix}, \qquad h = \begin{bmatrix} h_1 \\ h_2 \\ \vdots \end{bmatrix} \]

See Limits for the exact derivation of each limit type.

The complete QP

\[ \boxed{ \begin{aligned} \min_{\Delta q} \quad & \tfrac{1}{2}\,\Delta q^\top \left[\sum_i H_i + \lambda I\right] \Delta q + \left[\sum_i c_i\right]^\top \Delta q \\[4pt] \text{s.t.} \quad & G\,\Delta q \le h \end{aligned} } \]

OSQP solver

Torq uses OSQP via OsqpEigen. OSQP expects the standard form:

\[ \min_x \;\tfrac{1}{2}\,x^\top P\,x + q^\top x \quad\text{s.t.}\quad l \le A\,x \le u \]

The mapping from Torq's QPProblem is:

Torq OSQP
\(P\) (Hessian) \(P\) (upper triangle, sparse)
\(c\) \(q\)
\(G\) \(A\)
\(-\infty\) \(l\)
\(h\) \(u\)

The solver is warm-started across ticks: the sparsity pattern is set once and only the numerical values are updated on subsequent solves.

Manifold-aware integration

The solved \(\Delta q\) is a tangent vector. For joints with manifold structure (e.g. floating-base quaternions) Torq uses Pinocchio's pinocchio::integrate instead of simple addition:

\[ q_{\text{new}} = q \oplus \Delta q \]

For revolute and prismatic joints \(\oplus\) reduces to ordinary addition, so there is no overhead for fixed-base arms.
