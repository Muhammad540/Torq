Conventions and Notation

This page defines the mathematical notation, coordinate frame conventions, and manifold operations used throughout Torq and its documentation. Understanding these conventions is essential for correctly interpreting task errors, Jacobians, and constraint formulations.

Coordinate frames

Torq uses the following coordinate frame conventions, inherited from Pinocchio and MuJoCo:

World frame

The world frame is the fixed, global reference frame. All absolute positions and orientations are expressed relative to this frame. Gravity acts along the negative Z axis by default.

Body (local) frame

Each rigid body has a local frame attached to it. Frame transforms \(T_{\text{frame}}^{\text{world}}\) give the pose of a body frame in world coordinates.

Jacobian reference frames

Torq uses the LOCAL reference frame for frame Jacobians, consistent with the logarithmic map error formulation:

  • LOCAL (body frame): The Jacobian maps joint velocities to a twist expressed in the body's own frame. This is the natural choice when the error is computed via \(\log_6(T_{\text{cur}}^{-1} T_{\text{des}})\) because the logarithm produces a body-frame twist.
  • LOCAL_WORLD_ALIGNED: Used internally for some barrier computations (e.g. SelfCollisionBarrier) where the geometric quantities are naturally expressed in world-aligned coordinates.

SE(3) transform composition

Transforms compose by matrix multiplication. If \(T_A^W\) is the pose of frame A in world, and \(T_B^A\) is the pose of frame B in frame A:

\[ T_B^W = T_A^W \cdot T_B^A \]

  • Left-multiplication applies a transform in the world frame.
  • Right-multiplication applies a transform in the local frame.

Manifold operations

Robot configuration spaces are not always Euclidean. Floating-base robots use quaternions for orientation, which live on \(S^3\). Torq handles this using Pinocchio's manifold-aware operations.

Difference (\f$\ominus\f$)

The manifold difference computes a tangent vector between two configurations:

\[ \delta = q_2 \ominus q_1 = \text{pinocchio::difference}(\text{model}, q_1, q_2) \]

For revolute joints, this reduces to \(q_2 - q_1\). For quaternion joints, it computes the logarithmic map of the relative rotation. The result is always a vector in \(\mathbb{R}^{n_v}\).

Integration (\f$\oplus\f$)

Manifold integration applies a tangent vector to a configuration:

\[ q_{\text{new}} = q \oplus \Delta q = \text{pinocchio::integrate}(\text{model}, q, \Delta q) \]

For revolute/prismatic joints, \(\oplus\) reduces to ordinary addition. For floating-base quaternions, it uses proper quaternion exponential integration, avoiding gimbal lock and maintaining unit-norm constraints.

Why this matters

The QP solves for \(\Delta q \in \mathbb{R}^{n_v}\) (tangent space). After the solve, manifold integration maps this back to a valid configuration:

\[ q_{\text{new}} = q_{\text{current}} \oplus \Delta q \]

This is why \(n_q\) (configuration dimension) can differ from \(n_v\) (velocity/tangent dimension): a free-floating base has \(n_q = 7\) (position + quaternion) but \(n_v = 6\) (linear + angular velocity).

Task convention

Every task implements the first-order dynamics:

\[ J_i(q)\,\Delta q = -\alpha_i\,e_i(q) \]

The error \(e_i(q)\) is defined so that it is zero at the target and the system drives \(e \to 0\). The gain \(\alpha\) controls the convergence rate: with \(\alpha = 1\) the full error is corrected each tick (dead-beat); with \(\alpha < 1\) the error decays exponentially:

\[ \|e_n\| \approx \|e_0\| \cdot (1 - \alpha)^n \]

Limit convention

Limits produce inequality constraints of the form:

\[ G\,\Delta q \le h \]

These are hard constraints — the QP solver will never return a \(\Delta q\) that violates them. If no feasible solution exists, the solver reports infeasibility.

Barrier convention

Barriers enforce safety via Control Barrier Functions. A barrier function \(h(q) \ge 0\) defines the safe region. The CBF condition:

\[ \frac{\partial h}{\partial q} \dot{q} + \alpha\,h(q) \ge 0 \]

guarantees forward invariance of the safe set. In the QP, this becomes an inequality constraint (rows in \(G, h\)) and optionally a safe-displacement objective term.
