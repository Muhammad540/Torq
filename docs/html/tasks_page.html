Tasks

Tasks define the objective (cost) of the QP problem. Each task contributes a Hessian block \(H_i\) and a gradient vector \(c_i\) — see QP Formulation for how they compose.

This page derives the error and Jacobian for every concrete task type.

Common parameters

Every task inherits from torq::Task and exposes:

Parameter Symbol Range Effect
cost (weight) \(w\) \((0, \infty)\) Relative importance vs other tasks. Higher = prioritised.
gain \(\alpha\) \((0, 1]\) Convergence speed. 1.0 = dead-beat (full correction each tick). Lower values smooth the response.
lm_damping \(\mu\) \([0, \infty)\) Levenberg–Marquardt regularisation. Increase when target is unreachable to avoid velocity spikes.

All three are adjustable at runtime via setters on the task and via RobotSystem's IK tuning API.

FrameTask

Purpose: regulate the SE(3) pose of a named robot frame (typically the end-effector) to a desired target \(T_{\text{des}} \in SE(3)\).

Error (6-D)

The 6-D error is computed using the logarithmic map of the relative transform:

\[ e = \log_6\!\left(T_{\text{cur}}^{-1}\; T_{\text{des}}\right) = \begin{bmatrix} e_{\text{pos}} \\ e_{\text{ori}} \end{bmatrix} \in \mathbb{R}^6 \]

where \(\log_6 : SE(3) \to \mathfrak{se}(3)\) maps the relative transform into a twist in the local (body) frame.

This is a manifold-aware error: it correctly handles the topology of rotations without gimbal lock or discontinuities.

Jacobian (6×n_v)

The frame Jacobian in the LOCAL reference frame:

\[ J = {}^{\text{local}}J_{\text{frame}}(q) \]

computed by pinocchio::computeFrameJacobian with LOCAL reference frame. Using the local frame ensures consistency with the \(\log_6\) error.

Weight vector

The 6-D cost vector is \(w = [w_{\text{pos}}^{(3)},\; w_{\text{ori}}^{(3)}]\) where the first three components weight translational error and the last three weight rotational error. This allows independent prioritisation of position vs orientation tracking.

QP contribution

\[ W = \operatorname{diag}(w), \quad \bar{J} = W J, \quad \bar{e} = W(-\alpha\, e) \]

\[ H_{\text{frame}} = \bar{J}^\top \bar{J} + \mu\,(\bar{e}^\top\bar{e})\,I_{n_v}, \quad c_{\text{frame}} = -\bar{J}^\top \bar{e} \]

Setters

  • setPositionCost(double) / setPositionCost(Eigen::Vector3d)
  • setOrientationCost(double) / setOrientationCost(Eigen::Vector3d)
  • setGain(double), setLMDamping(double)
  • setTarget(pinocchio::SE3), setTargetFromConfiguration(Configuration)

When no target is set the task is inert — it contributes zero to the QP.

PostureTask

Purpose: regulate all actuated joint angles toward a reference posture \(q_{\text{ref}}\).

Error (n_v-D)

\[ e = q_{\text{ref}} \ominus q_{\text{cur}} \]

where \(\ominus\) is pinocchio::difference — the manifold-aware subtraction that produces a tangent vector. For revolute/prismatic joints this reduces to \(q_{\text{ref}} - q_{\text{cur}}\).

Floating-base root coordinates are excluded; only actuated joints contribute.

Jacobian (n_v×n_v)

The posture Jacobian is the identity matrix (after excluding root DOFs):

\[ J_{\text{posture}} = I_{n_v} \]

(More precisely, it is the identity on actuated DOFs and zero on root DOFs.)

Weight

A single scalar cost \(w_p\) is broadcast to all actuated joints: \(W = w_p\, I\). Typical value: \(10^{-3}\) — low priority regularisation.

QP contribution

\[ H_{\text{posture}} = w_p^2\, I_{n_v} + \mu\,(\bar{e}^\top\bar{e})\,I_{n_v}, \quad c_{\text{posture}} = -w_p^2\,\alpha\, e \]

When no target is set the task is inert .

DampingTask

Purpose: penalise all joint velocities equally, acting as viscous damping that brings the robot to rest.

Error

\[ e_{\text{damp}} = \mathbf{0} \in \mathbb{R}^{n_v} \]

The error is always zero — this task only contributes through its Jacobian.

Jacobian

\[ J_{\text{damp}} = I_{n_v} \]

QP contribution

With scalar cost \(w_d\) and zero error the LM term vanishes:

\[ H_{\text{damp}} = w_d^2\, I_{n_v}, \qquad c_{\text{damp}} = \mathbf{0} \]

This adds a quadratic penalty on \(\|\Delta q\|^2\) — the solver will minimise unnecessary motion. Typical cost: \(10^{-4}\).

No target needed; always active once constructed.

How tasks compose

The QP Hessian and gradient are the element-wise sum of all task contributions:

\[ P = H_{\text{frame}} + H_{\text{posture}} + H_{\text{damp}} + \lambda I, \quad c = c_{\text{frame}} + c_{\text{posture}} + c_{\text{damp}} \]

Because FrameTask typically has cost \(\gg\) PostureTask \(\gg\) DampingTask, the hierarchy is:

  1. Track the end-effector target (highest priority).
  2. Stay near the reference posture (medium priority, regularisation).
  3. Minimise overall motion (lowest priority, smoothing).

Adjusting costs shifts these priorities at runtime.
