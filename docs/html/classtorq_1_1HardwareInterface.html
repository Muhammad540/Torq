Detailed Description

Abstract hardware abstraction layer for any robot driver (simulation or real).

All control and kinematics code targets this interface. Swapping the concrete implementation (e.g. MujocoDriver → a real hardware driver) enables sim-to-real transfer without touching control logic.

Two "spaces" are exposed:

  • State space (getJointPositions / getJointVelocities): \(q \in \mathbb{R}^{n_q}\), \(\dot{q} \in \mathbb{R}^{n_v}\).
  • Actuator command space (setJointPositions / setJointVelocities, getCtrl): may differ from state space (e.g. underactuated systems, gear ratios).
