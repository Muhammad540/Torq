Class hierarchy

Core

  • torq::RobotSystem — owns HardwareInterface, KinematicsEngine, Controller. Single entry point for all robot operations.
  • torq::Controller — manages control modes (IDLE, JOINT_SPACE, TASK_SPACE), owns the IK solver and all task/limit instances, holds the IKConfig struct.
  • torq::InverseKinematics — assembles the QP from tasks and limits, delegates to OSQP.

Kinematics

  • torq::KinematicsEngine — loads URDF or MJCF, builds full and reduced Pinocchio models.
  • torq::Configuration — immutable snapshot of the robot state at a given \(q\). Provides frame transforms, Jacobians, and manifold-aware integration.

Tasks (QP objective)

  • torq::Task (abstract) — defines computeError(), computeJacobian(), computeQPObjective().
    • torq::FrameTask — regulate end-effector SE(3) pose.
    • torq::PostureTask — regulate joint angles toward a reference.
    • torq::DampingTask — penalize all joint velocities (viscous damping).

Limits (QP inequality constraints)

  • torq::Limit (abstract) — defines computeQPInequalities().
    • torq::VelocityLimit — \( -\Delta t \, v_{\max} \le \Delta q \le \Delta t \, v_{\max} \).
    • torq::ConfigurationLimit — keep \(q + \Delta q\) within \([q_{\min}, q_{\max}]\).

Hardware abstraction

  • torq::HardwareInterface (abstract) — connect, step, getJointPositions, setJointPositions, …
    • torq::MujocoDriver — RAII simulation backend wrapping mjModel / mjData.

GUI

  • torq::Gui — ImGui + GLFW + OpenGL3 docking interface. Panels for joint control, Cartesian jog, IK parameter tuning, and a 3D viewport rendering MuJoCo scenes.

Data flow per tick

  1. RobotSystem::update() reads joint state from HardwareInterface.
  2. The controller builds a Configuration from the current \(q\).
  3. Each Task computes its error and Jacobian from the Configuration.
  4. Each Limit computes its inequality rows from the Configuration.
  5. InverseKinematics::solve() assembles the QP and calls OSQP.
  6. The resulting \(\Delta q\) is integrated on the manifold, and the new \(q\) is written back to the hardware interface.

Adding a new robot

  1. Place model files (URDF/MJCF + meshes) under workspace/models/.
  2. Create workspace/<name>/src/main.cpp — fill a torq::RobotConfig and run the loop.
  3. Add workspace/<name>/CMakeLists.txt with target_link_libraries(<name> PRIVATE torq).
  4. Add add_subdirectory(workspace/<name>) to the root CMakeLists.txt.
